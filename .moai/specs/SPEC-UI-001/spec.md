## 요구사항 (Requirements)

### 모듈 1: 게임 엔진 (Game Engine)

#### R-001: 게임 초기화 (유비쿼터스)

시스템은 **항상** 게임 시작 시 Canvas를 초기화하고, 스네이크를 보드 중앙에 초기 길이(3 세그먼트)로 배치하며, 먹이를 랜덤 위치에 생성해야 한다.

#### R-002: 게임 루프 (유비쿼터스)

시스템은 **항상** requestAnimationFrame 기반 게임 루프를 실행하여 입력 처리, 상태 업데이트, 렌더링을 매 프레임 순차적으로 수행해야 한다.

#### R-003: 스네이크 이동 (이벤트 구동)

**WHEN** 게임 루프의 업데이트 틱이 발생하면 **THEN** 스네이크는 현재 방향으로 1 그리드 셀만큼 이동해야 한다.

#### R-004: 스네이크 성장 (이벤트 구동)

**WHEN** 스네이크의 머리가 먹이 위치와 동일한 그리드 셀에 도달하면 **THEN** 스네이크의 길이가 1 세그먼트 증가하고, 점수가 10점 증가하며, 새로운 먹이가 빈 셀에 생성되어야 한다.

#### R-005: 벽 충돌 감지 (이벤트 구동)

**WHEN** 스네이크의 머리가 게임 보드 경계를 벗어나면 **THEN** 게임 상태가 GAME_OVER로 전환되어야 한다.

#### R-006: 자기 충돌 감지 (이벤트 구동)

**WHEN** 스네이크의 머리가 자신의 몸체 세그먼트와 동일한 위치에 도달하면 **THEN** 게임 상태가 GAME_OVER로 전환되어야 한다.

#### R-007: 먹이 위치 유효성 (금지)

시스템은 스네이크의 몸체가 차지하고 있는 그리드 셀에 먹이를 생성**하지 않아야 한다**.

---

### 모듈 2: 입력 시스템 (Input System)

#### R-008: 키보드 방향 입력 (이벤트 구동)

**WHEN** 사용자가 화살표 키(ArrowUp, ArrowDown, ArrowLeft, ArrowRight)를 누르면 **THEN** 스네이크의 다음 이동 방향이 해당 방향으로 설정되어야 한다.

#### R-009: 반대 방향 전환 방지 (금지)

시스템은 스네이크의 현재 이동 방향과 정반대 방향으로의 방향 전환을 허용**하지 않아야 한다** (예: 오른쪽 이동 중 왼쪽으로 전환 불가).

#### R-010: 일시정지 토글 (이벤트 구동)

**WHEN** 사용자가 Space 또는 P 키를 누르면 **THEN** 게임 상태가 PLAYING과 PAUSED 사이에서 토글되어야 한다.

#### R-011: 모바일 터치 입력 (이벤트 구동, 선택)

**가능하면** 모바일 기기에서 스와이프 제스처를 감지하여 화살표 키와 동일한 방향 입력으로 변환 제공.

---

### 모듈 3: 점수 및 레벨 시스템 (Score & Level System)

#### R-012: 점수 계산 (이벤트 구동)

**WHEN** 스네이크가 먹이를 먹으면 **THEN** 현재 점수가 10점 증가해야 한다.

#### R-013: 레벨 진행 (이벤트 구동)

**WHEN** 현재 게임에서 먹은 먹이 수가 4의 배수에 도달하면 **THEN** 레벨이 1 증가해야 한다.

#### R-014: 속도 증가 (상태 구동)

**IF** 현재 레벨이 N이면 **THEN** 게임 속도는 기본 속도 x (1 + N x 0.15)로 계산되어야 한다.

#### R-015: 최고 점수 저장 (이벤트 구동)

**WHEN** 게임이 종료되고 현재 점수가 기존 최고 점수보다 높으면 **THEN** localStorage에 새로운 최고 점수를 저장해야 한다.

#### R-016: 최고 점수 로드 (이벤트 구동)

**WHEN** 게임이 초기화되면 **THEN** localStorage에서 이전 최고 점수를 불러와 UI에 표시해야 한다.

---

### 모듈 4: UI 렌더링 (UI Rendering)

#### R-017: 게임 보드 렌더링 (유비쿼터스)

시스템은 **항상** 게임 보드를 어두운 배경에 격자 패턴으로 렌더링하고, 스네이크를 네온 그린, 먹이를 붉은색 계열로 표시해야 한다.

#### R-018: HUD 표시 (유비쿼터스)

시스템은 **항상** 현재 점수, 현재 레벨, 최고 점수를 게임 화면 상단에 실시간으로 표시해야 한다.

#### R-019: 시작 메뉴 (상태 구동)

**IF** 게임 상태가 MENU이면 **THEN** "게임 시작" 버튼과 게임 제목을 표시해야 한다.

#### R-020: 게임오버 화면 (상태 구동)

**IF** 게임 상태가 GAME_OVER이면 **THEN** 최종 점수, 최고 기록 달성 여부, "다시 시작" 버튼을 표시해야 한다.

#### R-021: 일시정지 화면 (상태 구동)

**IF** 게임 상태가 PAUSED이면 **THEN** "일시정지" 오버레이와 "재개" 안내를 표시해야 한다.

#### R-022: 레벨업 애니메이션 (이벤트 구동, 선택)

**가능하면** 레벨 상향 시 레벨 숫자가 확대/축소되는 시각 효과 제공.

#### R-023: 게임오버 애니메이션 (이벤트 구동, 선택)

**가능하면** 게임 종료 시 스네이크와 보드가 페이드 아웃되는 시각 효과 제공.

---

### 모듈 5: 사운드 시스템 (Sound System)

#### R-024: 먹이 획득 효과음 (이벤트 구동, 선택)

**가능하면** 스네이크가 먹이를 먹을 때 효과음 재생 제공.

#### R-025: 게임오버 효과음 (이벤트 구동, 선택)

**가능하면** 게임 종료 시 게임오버 효과음 재생 제공.

#### R-026: 음소거 토글 (이벤트 구동, 선택)

**가능하면** 사용자가 음향을 음소거/해제할 수 있는 UI 컨트롤 제공.

---

### 모듈 6: 반응형 및 접근성 (Responsive & Accessibility)

#### R-027: Canvas 반응형 스케일링 (이벤트 구동)

**WHEN** 브라우저 창 크기가 변경되면 **THEN** Canvas가 화면 크기에 맞춰 적절히 스케일링되어야 한다.

#### R-028: 색상 대비 (유비쿼터스)

시스템은 **항상** WCAG 2.1 AA 기준의 색상 대비 비율(4.5:1 이상)을 충족해야 한다.

#### R-029: 키보드 접근성 (유비쿼터스)

시스템은 **항상** 마우스 없이 키보드만으로 모든 게임 기능에 접근 가능해야 한다.

## 명세 (Specifications)

### 게임 상수

| 상수 | 값 | 설명 |
|------|-----|------|
| GRID_SIZE | 20 | 그리드 셀 크기 (px) |
| BOARD_WIDTH | 20 | 보드 가로 셀 수 |
| BOARD_HEIGHT | 20 | 보드 세로 셀 수 |
| INITIAL_SNAKE_LENGTH | 3 | 초기 스네이크 길이 |
| BASE_SPEED | 150 | 기본 이동 간격 (ms) |
| POINTS_PER_FOOD | 10 | 먹이당 점수 |
| FOODS_PER_LEVEL | 4 | 레벨업 필요 먹이 수 |
| SPEED_MULTIPLIER | 0.15 | 레벨당 속도 증가율 |

### 게임 상태 머신

```
MENU --[시작 버튼]--> PLAYING
PLAYING --[Space/P]--> PAUSED
PAUSED --[Space/P]--> PLAYING
PLAYING --[충돌]--> GAME_OVER
GAME_OVER --[다시 시작]--> PLAYING
```

### 데이터 구조

**Position**: x(number), y(number) - 그리드 좌표

**Direction**: UP, DOWN, LEFT, RIGHT - 이동 방향

**GameState**: MENU, PLAYING, PAUSED, GAME_OVER - 게임 상태 열거형

**Snake**: segments(Position[]), direction(Direction), nextDirection(Direction)

**Food**: position(Position)

### 속도 공식

```
moveInterval = BASE_SPEED / (1 + level * SPEED_MULTIPLIER)
```

- 레벨 1: 150 / 1.15 = 130ms
- 레벨 2: 150 / 1.30 = 115ms
- 레벨 3: 150 / 1.45 = 103ms
- 레벨 5: 150 / 1.75 = 85ms
